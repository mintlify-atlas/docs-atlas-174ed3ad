---
title: "Quick start"
description: "Get a Minestom server running with a working player spawn in about five minutes."
---

This guide walks you through building a minimal Minestom server: one that accepts connections, places players in a world, and starts ticking. All code is derived from the [demo application](https://github.com/minestom/minestom/tree/main/demo/src/main/java/net/minestom/demo).

## Prerequisites

- Java 21 or later
- Gradle or Maven (this guide uses Gradle Kotlin DSL)
- Minestom added as a dependency — see [Installation](/installation) if you haven't done this yet

---

<Steps>

<Step title="Add the dependency">

Add Minestom to your `build.gradle.kts`. Replace `VERSION` with the latest release from the [GitHub releases page](https://github.com/minestom/minestom/releases).

```kotlin
repositories {
    mavenCentral()
}

dependencies {
    implementation("net.minestom:minestom:VERSION")
}
```

See [Installation](/installation) for full setup instructions including Maven and the Java module system.

</Step>

<Step title="Initialize the server">

Call `MinecraftServer.init()` before registering any game logic. This bootstraps all internal managers and registries.

```java
import net.minestom.server.MinecraftServer;
import net.minestom.server.Auth;

public class Main {
    public static void main(String[] args) {
        MinecraftServer minecraftServer = MinecraftServer.init(new Auth.Offline());
        
        // Register your game logic here (instances, events, commands)

        minecraftServer.start("0.0.0.0", 25565);
    }
}
```

<Note>
  `MinecraftServer.init()` must be called before accessing any manager (e.g., `MinecraftServer.getInstanceManager()`). Calling managers before `init()` will throw a `NullPointerException` because the internal `ServerProcess` has not been created yet.

  Passing `new Auth.Offline()` disables online-mode authentication. Players can connect without a valid Mojang account. Use `Auth.Online()` (or configure Velocity/BungeeCord proxy auth) for production.
</Note>

</Step>

<Step title="Create an instance">

An instance is Minestom's equivalent of a world. You must create at least one before players can join. Use `InstanceManager` to create an `InstanceContainer`, then assign a generator to define the terrain.

```java
import net.minestom.server.instance.InstanceContainer;
import net.minestom.server.instance.InstanceManager;
import net.minestom.server.instance.LightingChunk;
import net.minestom.server.instance.block.Block;

InstanceManager instanceManager = MinecraftServer.getInstanceManager();

InstanceContainer instanceContainer = instanceManager.createInstanceContainer();
instanceContainer.setGenerator(unit -> {
    unit.modifier().fillHeight(0, 40, Block.STONE);
});
instanceContainer.setChunkSupplier(LightingChunk::new);
```

`setGenerator` is called for each chunk as it is loaded. The example fills blocks from y=0 to y=40 with stone, producing a flat surface at y=40. `LightingChunk` enables proper block lighting calculations.

</Step>

<Step title="Handle player join">

Listen for `AsyncPlayerConfigurationEvent` to assign a spawning instance to each connecting player, and `PlayerSpawnEvent` to set their initial game mode and spawn position.

```java
import net.minestom.server.coordinate.Pos;
import net.minestom.server.entity.GameMode;
import net.minestom.server.event.GlobalEventHandler;
import net.minestom.server.event.player.AsyncPlayerConfigurationEvent;
import net.minestom.server.event.player.PlayerSpawnEvent;

GlobalEventHandler eventHandler = MinecraftServer.getGlobalEventHandler();

eventHandler.addListener(AsyncPlayerConfigurationEvent.class, event -> {
    event.setSpawningInstance(instanceContainer);
    event.getPlayer().setRespawnPoint(new Pos(0, 40f, 0));
});

eventHandler.addListener(PlayerSpawnEvent.class, event -> {
    event.getPlayer().setGameMode(GameMode.CREATIVE);
});
```

<Warning>
  Every player **must** be assigned a spawning instance in `AsyncPlayerConfigurationEvent`. If `setSpawningInstance` is not called, the player will be kicked immediately after the configuration phase with a connection error.
</Warning>

</Step>

<Step title="Start the server">

With instances and events registered, call `start()` to begin accepting connections.

```java
minecraftServer.start("0.0.0.0", 25565);
```

`start()` blocks the calling thread and begins the tick loop. The server is now reachable at `localhost:25565`.

</Step>

</Steps>

## Complete example

Putting the above together into a single `Main.java`:

```java
import net.minestom.server.Auth;
import net.minestom.server.MinecraftServer;
import net.minestom.server.coordinate.Pos;
import net.minestom.server.entity.GameMode;
import net.minestom.server.event.GlobalEventHandler;
import net.minestom.server.event.player.AsyncPlayerConfigurationEvent;
import net.minestom.server.event.player.PlayerSpawnEvent;
import net.minestom.server.instance.InstanceContainer;
import net.minestom.server.instance.InstanceManager;
import net.minestom.server.instance.LightingChunk;
import net.minestom.server.instance.block.Block;

public class Main {
    public static void main(String[] args) {
        MinecraftServer minecraftServer = MinecraftServer.init(new Auth.Offline());

        InstanceManager instanceManager = MinecraftServer.getInstanceManager();
        InstanceContainer instanceContainer = instanceManager.createInstanceContainer();
        instanceContainer.setGenerator(unit -> {
            unit.modifier().fillHeight(0, 40, Block.STONE);
        });
        instanceContainer.setChunkSupplier(LightingChunk::new);

        GlobalEventHandler eventHandler = MinecraftServer.getGlobalEventHandler();

        eventHandler.addListener(AsyncPlayerConfigurationEvent.class, event -> {
            event.setSpawningInstance(instanceContainer);
            event.getPlayer().setRespawnPoint(new Pos(0, 40f, 0));
        });

        eventHandler.addListener(PlayerSpawnEvent.class, event -> {
            event.getPlayer().setGameMode(GameMode.CREATIVE);
        });

        minecraftServer.start("0.0.0.0", 25565);
    }
}
```
