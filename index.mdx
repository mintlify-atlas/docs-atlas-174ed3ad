Minestom is a Java library that lets you build Minecraft servers from the ground up. Unlike traditional server software, Minestom ships with **no world generation, no default gameplay, and no built-in commands** — giving you complete control over every aspect of the player experience.

It targets Minecraft **1.21.1** and is designed for developers who need performance, flexibility, and direct access to the server's internals.

## Why Minestom?

Minestom is not a drop-in replacement for Bukkit or Paper. It is a **blank slate** — ideal for minigame servers, game platforms, and any use case where standard Minecraft gameplay would get in the way.

<CardGroup cols={2}>
  <Card title="Zero overhead by default" icon="gauge-high">
    No plugins to disable, no world to pregenerate. You only pay for what you implement.
  </Card>
  <Card title="Multi-threaded ticking" icon="microchip">
    The `ThreadDispatcher` distributes instance and entity ticks across configurable thread pools.
  </Card>
  <Card title="Hierarchical event system" icon="diagram-project">
    `EventNode` trees let you filter, scope, and compose event listeners without global pollution.
  </Card>
  <Card title="Type-safe commands" icon="terminal">
    The command builder provides argument parsing, tab-completion, and execution conditions in a fluent API.
  </Card>
  <Card title="Adventure integration" icon="palette">
    Full support for the Adventure library — rich text, boss bars, titles, sounds, and resource packs.
  </Card>
  <Card title="Dynamic registries" icon="database">
    Register custom dimensions, biomes, enchantments, and more at runtime without datapacks.
  </Card>
</CardGroup>

## Get started in four steps

<Steps>
  <Step title="Add the dependency">
    Add Minestom to your `build.gradle.kts` or `pom.xml`. See [Installation](/installation) for coordinates.
  </Step>
  <Step title="Initialize the server">
    Call `MinecraftServer.init()` to bootstrap all managers and registries.
  </Step>
  <Step title="Register your game logic">
    Add event listeners, commands, instances, and entities between `init()` and `start()`.
  </Step>
  <Step title="Start listening">
    Call `minecraftServer.start("0.0.0.0", 25565)` to begin accepting connections.
  </Step>
</Steps>
